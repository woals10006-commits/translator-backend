==================================================
  중국어→한국어 번역 프로그램 - 새 컴퓨터 설치 안내
  (IntelliJ + VS Code 방식 / 완전 초보자용)
==================================================

이 프로그램은 워드(.docx) 파일을 올리면 중국어를 한국어로
번역해서 다시 워드 파일로 내려주는 프로그램입니다.

이슈단어 프로그램과 달리, 이건 설치할 게 여러 개 있습니다.
아래 순서를 "위에서 아래로 하나씩" 그대로 따라 하세요.


==================================================
 [ 프로그램 구성 ]  두 덩어리로 되어 있습니다
==================================================

 1) translator-backend   <- 번역 엔진 (자바 / IntelliJ로 실행)
 2) translator-frontend  <- 화면 (React / VS Code로 실행)

 두 폴더를 모두 새 컴퓨터로 복사해야 합니다.
 (예: C:\translator-backend , C:\translator-frontend 에 두는 것을 추천)


==================================================
 [ 설치할 것 요약 ]  총 4개 + API 키
==================================================

 (1) JDK 17        - 자바. 백엔드를 돌리는 데 필수
 (2) IntelliJ      - 백엔드 실행용 프로그램(버튼으로 실행)
 (3) Node.js       - 프론트엔드를 돌리는 데 필수
 (4) VS Code       - 프론트엔드 실행/편집용 프로그램
 (5) API 키        - Anthropic(Claude) 키. 번역할 때 돈이 나감

 ※ 순서대로 (1)→(2)→(3)→(4) 설치하고, 그 다음 실행합니다.


==================================================
 [ 1 ] JDK 17 설치  (백엔드용 자바)  ★중요★
==================================================

 이 프로그램은 자바가 정확히 이 위치에 있길 기대합니다:
      C:\Program Files\Java\jdk-17
 그래서 "Oracle JDK 17"을 기본 설정으로 깔면 딱 맞습니다.

 1. Edge 브라우저를 엽니다.
 2. 주소창에 입력:
      https://www.oracle.com/java/technologies/javase/jdk17-archive-downloads.html
    (또는 검색창에 "oracle jdk 17 windows download" 검색)
 3. "Windows x64 Installer" (파일명 예: jdk-17.0.x_windows-x64_bin.exe) 다운로드
 4. 다운로드한 exe 파일 더블클릭 → "다음(Next)"만 계속 눌러 설치
    (설치 위치를 바꾸지 마세요. 기본값 그대로 두어야 위 경로와 맞습니다.)

 [ 설치 후 경로 확인 - 매우 중요 ]
 - 파일 탐색기에서 C:\Program Files\Java\ 폴더를 열어봅니다.
 - 안에 "jdk-17" 폴더가 있으면 완벽합니다. 그대로 진행하세요.
 - 만약 폴더 이름이 "jdk-17.0.12" 처럼 뒤에 숫자가 붙어 있으면,
   나중에 [문제해결] 맨 아래 "자바 경로 안 맞을 때"를 보고 한 줄만 고치면 됩니다.
   (지금은 그냥 넘어가도 됩니다.)


==================================================
 [ 2 ] IntelliJ IDEA 설치  (백엔드 실행 프로그램)
==================================================

 1. 주소창에 입력:  https://www.jetbrains.com/idea/download
 2. 아래로 스크롤해서 "IntelliJ IDEA Community Edition" (무료) 를 받습니다.
    ※ 위쪽의 "Ultimate"는 유료입니다. 아래쪽 "Community Edition"(무료)로 받으세요.
 3. 다운로드한 exe 더블클릭 → "다음"만 계속 눌러 설치 → "Finish"
 4. 설치가 끝나면 IntelliJ를 한 번 실행해 둡니다.
    (처음 실행 시 테마 고르라고 나오면 아무거나 골라도 됩니다.)


==================================================
 [ 3 ] Node.js 설치  (프론트엔드용)
==================================================

 1. 주소창에 입력:  https://nodejs.org
 2. 초록색 버튼 중 "LTS" 라고 적힌 버전을 다운로드합니다.
    (LTS = 안정 버전. 이걸 받으세요.)
 3. 다운로드한 설치 파일(.msi) 더블클릭 → "Next"만 계속 → 설치
    (중간 체크박스들은 기본값 그대로 두면 됩니다.)


==================================================
 [ 4 ] VS Code 설치  (프론트엔드 실행/편집 프로그램)
==================================================

 1. 주소창에 입력:  https://code.visualstudio.com
 2. 파란색 "Download for Windows" 버튼 클릭
 3. 다운로드한 exe 더블클릭 → "동의합니다" 체크 → "다음" 계속 → 설치


==================================================
 [ 5 ] API 키 준비  (번역에 돈이 나가는 부분)
==================================================

 번역은 Claude(Anthropic) AI를 사용하므로 "API 키"가 필요합니다.

 - 이미 예전 컴퓨터에서 쓰던 키가 있으면 그 키를 그대로 씁니다.
   (키는 "sk-ant-..." 로 시작하는 긴 문자열입니다.)
 - 키가 없거나 새로 만들려면:
     1. https://console.anthropic.com 접속 → 로그인
     2. 왼쪽 메뉴 "API Keys" → "Create Key" → 키 복사
 - 이 키는 남에게 보여주면 안 됩니다. (돈이 나가는 키라서요.)

 ※ 비용 참고: 100화(약 1만 단락) 번역 시 대략 $6, 시간은 약 48분.
   console.anthropic.com 에서 "월 지출 한도"를 걸어두면 안심입니다.


==================================================
 [ 6 ] 백엔드 실행하기  (IntelliJ)
==================================================

 1. IntelliJ 실행 → 첫 화면에서 "Open" 클릭
 2. C:\translator-backend 폴더를 선택 → "OK"
 3. 오른쪽 아래에 뭔가 로딩되는 표시(Gradle 어쩌고)가 뜨면
    "끝날 때까지" 몇 분 기다립니다. (처음 한 번만 오래 걸림)

 4. [ API 키 넣기 ] - 실행 전에 꼭 해야 함
    가장 쉬운 방법: 윈도우 환경변수로 넣기
     a. 윈도우 시작 버튼 옆 검색창에 "환경 변수" 입력
        → "시스템 환경 변수 편집" 클릭
     b. 아래쪽 "환경 변수(N)..." 버튼 클릭
     c. 위쪽 "사용자 변수"에서 "새로 만들기(N)..." 클릭
     d. 변수 이름:  CLAUDE_API_KEY
        변수 값:    (복사해둔 sk-ant-... 키 붙여넣기)
     e. "확인" → "확인" → "확인" 으로 모두 닫기
     f. ★중요★ IntelliJ가 켜져 있었다면 "완전히 종료했다가 다시 켜세요".
        (안 그러면 새 환경변수를 인식 못 합니다.)

 5. [ 실행 ]
     a. IntelliJ 왼쪽 파일 목록에서 아래 파일을 찾아 더블클릭해 엽니다:
          src > main > java > com.translator > TranslatorApplication
     b. 코드 안 10번째 줄쯤 "public static void main(...)" 옆에
        초록색 ▶ (삼각형) 버튼이 있습니다. 그걸 클릭 → "Run" 선택
     c. 아래쪽 검은 창에 로그가 주르륵 뜨고,
        "Started TranslatorApplication ..." 이 보이면 성공입니다.
     d. 이 창은 "끄지 말고 켜둔 채로" 둡니다. (번역하는 동안 계속 켜져 있어야 함)

  => 백엔드가 http://localhost:8080 에서 돌고 있는 상태가 됩니다.


==================================================
 [ 7 ] 프론트엔드 실행하기  (VS Code)
==================================================

 1. VS Code 실행 → 상단 메뉴 File → Open Folder
 2. C:\translator-frontend 폴더 선택 → 열기
    (혹시 "이 폴더의 작성자를 신뢰합니까?" 나오면 "Yes, I trust" 클릭)

 3. 상단 메뉴 Terminal → New Terminal 클릭
    (화면 아래쪽에 명령어 입력창이 생깁니다.)

 4. 그 입력창에 아래를 치고 Enter (처음 한 번만 하면 됨):
        npm install
    => 설치가 끝날 때까지 기다립니다. (몇 분 걸릴 수 있음)

 5. 이어서 아래를 치고 Enter:
        npm run dev
    => "Local: http://localhost:5173" 같은 주소가 나옵니다.
    => 이 터미널도 "켜둔 채로" 둡니다.

 6. 그 http://localhost:5173 주소를 마우스로 Ctrl+클릭 하거나,
    Edge 주소창에 직접 쳐서 엽니다.


==================================================
 [ 8 ] 사용하기
==================================================

 1. 브라우저에 번역기 화면이 뜹니다.
 2. 번역할 워드(.docx) 파일을 올립니다.
 3. "번역할 화 수"를 입력합니다.
 4. "번역 시작"을 누르면 진행률이 올라가고,
    끝나면 번역된 파일이 자동으로 다운로드됩니다.

 ※ 번역 도중에는 IntelliJ(백엔드)와 VS Code 터미널(프론트),
   둘 다 켜져 있어야 합니다. 하나라도 끄면 멈춥니다.


==================================================
 [ 다음부터 켤 때 ]  (설치 다 한 뒤, 매번 쓰는 순서)
==================================================

 설치는 처음 한 번만. 그 다음부터는 이 순서만 반복하면 됩니다:

 1. IntelliJ 열기 → TranslatorApplication 의 ▶ 버튼으로 Run
    (Started ... 뜰 때까지 기다리기)
 2. VS Code 열기 → Terminal → New Terminal → "npm run dev" 입력
    (npm install 은 처음 한 번만 하면 되고, 이후엔 안 해도 됨)
 3. http://localhost:5173 접속해서 사용


==================================================
 [ 문제 해결 ]
==================================================

Q. 백엔드 실행 시 "invalid x-api-key" / "authentication_error" 오류가 뜬다.
 → API 키(CLAUDE_API_KEY)가 안 들어갔거나 틀렸습니다.
   [5],[6-4] 다시 확인하고, 환경변수 넣은 뒤 IntelliJ를 껐다 켰는지 확인하세요.

Q. 백엔드 실행 시 자바 관련 오류 / "jdk-17" 경로를 못 찾는다고 나온다.
 → (자바 경로 안 맞을 때)
   1. C:\Program Files\Java\ 폴더를 열어 실제 자바 폴더 이름을 확인합니다.
      (예: jdk-17.0.12)
   2. C:\translator-backend\gradle.properties 파일을 VS Code나 메모장으로 엽니다.
   3. 안에 이렇게 되어 있는 줄을:
          org.gradle.java.home=C:\\Program Files\\Java\\jdk-17
      실제 폴더 이름으로 고칩니다. 예:
          org.gradle.java.home=C:\\Program Files\\Java\\jdk-17.0.12
      (역슬래시는 반드시 두 개 \\ 로!)
   4. 저장 후 IntelliJ에서 다시 Run.

Q. 프론트엔드 "npm : 용어가 아닙니다" 라고 나온다.
 → Node.js가 설치 안 됐거나, VS Code를 Node 설치 전에 켠 겁니다.
   [3] Node.js 설치 확인 → VS Code를 껐다 다시 켜고 다시 시도하세요.

Q. 브라우저 화면은 뜨는데 "번역 시작"을 눌러도 안 된다 / 업로드 실패.
 → 백엔드(IntelliJ)가 안 켜져 있는 겁니다. [6] 을 먼저 실행하세요.
   IntelliJ 로그에 "Started TranslatorApplication" 이 떠 있어야 합니다.

Q. 포트 8080 / 5173 이 이미 사용 중이라고 나온다.
 → 이전에 켜둔 창이 남아있는 겁니다. 컴퓨터를 재부팅하고 다시 [6],[7] 하세요.
